# helper function
def plot_degree_distribution(G):
    """Computes the node degree distribution
    
    Keyword arguments:
    degs -- list of all degrees
    G -- the graph we want to plit the degree distribution of
    
    We go through the nodes in the graph G and add each degree, while coutnting the number of node with such a degree.
    It is possible to have some degrees with 0 nodes having this degree.
    
    return a table containing each degree and how many nodes are of such a degree
    """
    degs = list()
    for n in G.nodes():
        deg = G.degree(n)
        if deg >= len(degs):
            degs = degs + [0] * (deg - len(degs) + 1)
        degs[deg] += 1
    return degs

Exercise 2.4: Short paths & Small-worlds¶

If we think about the fact that we have a giant component that is not connected to other components in any way, it is quite obvious that network1 is in fact not a small world, as there would be no way to get to another component from the giant component in six degrees.

Nevertheless, let's plot the shortest path length between the nodes of the graph.

In [7]:
def short_path(graph, max_len, giant):
    shorts = nx.all_pairs_shortest_path_length(graph, max_len) #/!\ HEAVY
    short_distrib = []
    let_six = 0 #number of node pair with a shortest path lesser than 6
    shortest_path_count = 0
    # Estimates the number of shortest paths between two nodes
    for component in giant:
        node_count = len(component.nodes())
        shortest_path_count += (node_count**2 - node_count)/2

    # Look at length of the shortest paths in shorts and create a list of how many paths are of length n
    for n in shorts.values():
        let_six += len(n.values())
        for deg in n.values():
            if deg >= len(short_distrib):
                short_distrib = short_distrib + [0] * (deg - len(short_distrib) + 1)
            short_distrib[deg] += 1 
    return short_distrib, let_six, shortest_path_count

Exercise 2.6: Network identification¶

We believe that the first network (network1) is the subgraph of routers comprising the Internet and that the second network (network2) is the network of routes in New York.

Because the nodes in one of the graphs are supposed to be intersections, it is clearly impossible to have a node of degree more greater than even 10 (imagine 1500), simply because an intersection with a thousand roads going out of it is impossible due to physical limitations in the real world. Thus, it is quite evident that the graph that has a maximum node degree of 8 is a network of routes. Moreover, the second graph is clearly not a small world, as it is expected from physical roads while the first one is, as it is expected for router network.

It is also not a problem that the network of routers is not a small world, seeing as it is only a subgraph and thus could mean that nodes not chosen in this graph connect each component with the giant component in some way. But seeing as these nodes are outside of the graph, the edges are not presented in network1.
